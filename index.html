How it works: Draws a linear decision boundary between diabetic and non-diabetic classes. Uses sigmoid function to output a probability score (0–1). Classified as diabetic if probability > 0.5.
✓
Strengths: Simple, interpretable, fast training. Coefficients directly show which features are most important.
⚠
Limitation: Assumes a linear decision boundary — may underperform if the true boundary is complex or curved.
🌳
How it works: Asks if-else questions (e.g. Glucose > 127?) to split data into groups. Each leaf gives a prediction. Depth is limited to 5 to prevent overfitting.
⚠
Overfitting risk: At depth=20, training accuracy was 100% but validation dropped to 68% — memorized training data instead of learning patterns.
✓
Fix applied: max_depth=5 found via validation curve analysis — balances training (82%) and validation (77%) accuracy.
🌲
How it works: Builds 100 diverse decision trees, each trained on a random data subset (bagging). Final answer is the majority vote of all trees.
✓
Why it's best: Averaging 100 trees reduces variance and overfitting. Highest AUC (0.845) and most stable cross-validation scores.
🔑
Feature importance: Glucose > BMI > Age > DiabetesPedigreeFunction — confirmed across all trees.
